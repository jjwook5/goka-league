# ⛳ 고카리그 (GOKA LEAGUE) 시스템 개발 명세서 Ver 1.0

본 문서는 **고카리그(GOKA LEAGUE) Ver 1.0** 의 아키텍처, 데이터베이스 설계, 비즈니스 로직 및 UI/UX 구조를 기록한 공식 명세서입니다.  
향후 기능 고도화 및 유지보수 시 본 문서를 기준으로 처리합니다.

---

## 1. 시스템 개요 (System Overview)

| 항목 | 내용 |
|------|------|
| **시스템명** | 고카리그 (GOKA LEAGUE) Ver 1.0 |
| **대상** | 반기별 승강제 골프 동호회 운영 관리자 및 회원 |
| **배포 환경** | GitHub + Vercel (정적 호스팅, 자동 배포) |
| **데이터베이스** | Supabase (PostgreSQL) |
| **백엔드 미들웨어** | **없음** — 프론트엔드가 Supabase REST API 직접 호출 |
| **회원 접속 URL** | `golfriend.vercel.app` |
| **관리자 접속 URL** | `golfriend.vercel.app/admin/` |

### 1.1. 주요 핵심 기능

1. **시즌제 승강제 연동** — 상/하반기 단위 독립 시즌 운영 및 1부/2부/3부 리그 관리
2. **AI 스코어 추출 연동** — 스코어카드 이미지를 AI가 분석하여 JSON 변환 → 붙여넣기 기반 입력
3. **신페리오 정산 엔진** — 무작위 제외홀 추첨 + 소수 둘째자리 정밀 핸디캡 연산
4. **운세 분석 (Luck Factor)** — 추첨 운에 의한 변수를 실력 점수와 비교 계량화
5. **관리자 통합 관리** — 회원/코스/시즌/대회 CRUD를 관리자 화면에서 직접 처리
6. **모바일 퍼스트 PWA** — 홈 화면 설치 시 네이티브 앱처럼 동작 (주소창 없음)

### 2.4. Supabase REST API 호출 패턴

```javascript
// 공통 헤더 설정
const SB_H  = { 'apikey': KEY, 'Authorization': 'Bearer ' + KEY };
const SB_HW = { ...SB_H, 'Content-Type': 'application/json', 'Prefer': 'return=minimal' };

// 조회 (GET)
fetch(`${SUPABASE_URL}/rest/v1/{테이블}?select=*{필터}`, { headers: SB_H })

// 삽입 (POST)
fetch(`${SUPABASE_URL}/rest/v1/{테이블}`, { method:'POST', headers: SB_HW, body: JSON.stringify(rows) })

// 수정 (PATCH)
fetch(`${SUPABASE_URL}/rest/v1/{테이블}?{필터}`, { method:'PATCH', headers: SB_HW, body: JSON.stringify(fields) })

// 삭제 (DELETE)
fetch(`${SUPABASE_URL}/rest/v1/{테이블}?{필터}`, { method:'DELETE', headers: SB_H })

// 한국어 필터값은 반드시 encodeURIComponent() 적용
// 예: `전화번호=eq.${encodeURIComponent('010-1000-0001')}`
```

## 3. 데이터베이스 설계 (Supabase PostgreSQL)

### 3.1. `tb_member` — 회원 마스터

| 컬럼명 | 타입 | 설명 |
|--------|------|------|
| `순번` | int | 자동 증가 |
| `전화번호` | text | **PK** — 실제로는 회원번호로 사용 (형식 아래 참고) |
| `성명` | text | 실명 |
| `닉네임` | text | 선택 입력 |
| `소속리그` | text | 1부 / 2부 / 3부 |
| `등급` | text | 일반 / 스텝 |
| `활동상태` | text | **활동중** / 중지 (소프트 삭제) |

**회원번호 채번 규칙** (필드명은 `전화번호` 유지)

| 성별 | 형식 | 예시 |
|------|------|------|
| 남성 | `010-1000-XXXX` | `010-1000-0001` |
| 여성 | `010-2000-XXXX` | `010-2000-0002` |

- 뒷 4자리(XXXX)는 **남녀 공통 최댓값 + 1** (zero-padding 4자리)
- 관리자 화면에서 성별 선택 시 자동 채번 표시 (수동 입력 불가)

---

### 3.2. `tb_Course` — 코스 마스터

| 컬럼명 | 타입 | 설명 |
|--------|------|------|
| `구장명` | text | **PK(복합)** |
| `코스명` | text | **PK(복합)** |
| `P1`~`P9` | int | 홀별 기준타수 (3 / 4 / 5) |
| `사용여부` | text | **Y** / N (소프트 삭제) |

- `사용여부 = 'Y'` 인 코스만 대회 매핑 화면 및 신규 대회 생성 시 표시
- 코스 삭제 시 물리 삭제 없이 `사용여부 = 'N'` 으로 처리하여 과거 기록 무결성 유지

---

### 3.3. `tb_Match_Episodes` — 대회 에피소드

| 컬럼명 | 타입 | 설명 |
|--------|------|------|
| `대회번호` | bigint | `YYYYMMDD` 형식 날짜 채번 |
| `대회명` | text | 대회 이름 |
| `날짜` | date | 라운딩 날짜 |
| `구장명` | text | 사용 골프장 |
| `코스` | text | 9홀 코스명 (전반/후반 각각 1행) |
| `제외홀` | text | 신페리오 추첨 결과 (쉼표 구분, 예: `2,5,7`) |
| `완료구분` | text | **진행중** / 완료 |

- 하나의 대회에 코스 수만큼 행이 생성 (일반적으로 구장 내 코스 수 = 2~3행)
- `완료구분 = '완료'` 처리 시 관리자 세션 드롭다운에서 제외

### 3.4. `tb_Record` — 성적 트랜잭션 (핵심)

| 컬럼명 | 타입 | 설명 |
|--------|------|------|
| `순번` | int | 자동 증가 |
| `대회번호` | bigint | **PK(복합)** |
| `전화번호` | text | **PK(복합)** — tb_member FK |
| `구장명` | text | |
| `전반코스` | text | 전반 9홀 코스명 |
| `후반코스` | text | 후반 9홀 코스명 |
| `성명` | text | 대회 시점 성명 스냅샷 |
| `닉네임` | text | 대회 시점 닉네임 스냅샷 |
| `조번호` | text | 예: 1조, 2조 |
| `소속팀` | text | 백팀 / 청팀 (대회마다 재배정) |
| `소속리그` | text | 대회 시점 리그 스냅샷 |
| `H1`~`H18` | int | **오버파 표기법** 홀별 스코어 |
| `합계` | numeric | 그로스 스코어 (기준타 + 오버파 합산) |
| `NP_Handicap` | numeric | 신페리오 핸디캡 (소수 둘째자리) |
| `Net_Score` | numeric | 그로스 − NP_Handicap (소수 둘째자리) |
| `Luck_Factor` | numeric | 운세 지수 (소수 둘째자리) |

**스코어 오버파 표기법**

| 결과 | DB 저장값 |
|------|----------|
| 이글 | `-2` |
| 버디 | `-1` |
| 파 | `0` |
| 보기 | `1` |
| 더블보기 | `2` |
| 트리플 이상 | `3` 이상 |

### 3.5. `tb_Season` — 시즌 마스터

| 컬럼명 | 타입 | 설명 |
|--------|------|------|
| `시즌ID` | text | PK, 예: `S2026_1` |
| `시즌명` | text | 예: `2026년 상반기` |
| `시작_대회번호` | bigint | 시즌 첫 대회번호 (YYYYMMDD) |
| `종료_대회번호` | bigint | 시즌 마지막 대회번호 / **NULL = 진행중** |
| `진행상태` | text | **진행중** / 종료 |

- 시즌 범위 필터: `대회번호 >= 시작_대회번호 AND 대회번호 <= 종료_대회번호`
- 진행중 시즌은 `종료_대회번호 IS NULL` → 별도 처리 필요

## 4. 핵심 비즈니스 로직

### 4.1. 신페리오 제외홀 추첨

- **실행 시점:** 관리자 화면 → 대회관리 → 3단계 "🎲 제외홀 추첨 & 신페리오 정산" 버튼
- **알고리즘:** `tb_Course` 마스터에서 코스별 P3/P4/P5 홀 번호를 분류 후, 파별 각 1개 무작위 추출 (9홀당 3개 = 18홀 총 6개 제외홀)
- **저장:** `tb_Match_Episodes.제외홀` 에 쉼표 구분 문자열로 저장
- **재추첨 가능:** 대회 종료 전까지 버튼을 여러 번 눌러 재추첨 가능

### 4.2. 신페리오 핸디캡 (NP Engine)

모든 계산 로직은 **브라우저 측 JavaScript** 에서 실행됩니다 (서버 미들웨어 없음).

```
① 언더파 보정 (Cap)
   각 홀 오버파 값 = Math.max(0, 홀 스코어)  ← 버디(-1) 등은 0으로 처리

② 제외홀 적용
   overParSelected = 추첨된 12개 홀의 보정 오버파 합산
   overParTotal    = 전체 18홀 보정 오버파 합산

③ NP_Handicap 공식
   NP_Handicap = Math.round(overParSelected × 1.5 × 0.8 × 100) / 100
                 → 소수 둘째자리

④ 그로스 스코어
   합계 = 코스 기준타수(36) × 2(전후반) + 18홀 오버파 실제값 합산
   ※ 언더파는 보정 없이 실제 음수 그대로 합산

⑤ Net_Score
   Net_Score = Math.round((합계 - NP_Handicap) × 100) / 100
               → 소수 둘째자리

⑥ Luck_Factor
   Luck_Factor = Math.round((NP_Handicap - overParTotal × 0.8) × 100) / 100
                 → 소수 둘째자리
                 양수: 행운아 (핸디캡을 많이 받음)
                 음수: 불운왕 (핸디캡을 적게 받음)
```

### 4.3. 승강제 리그 순위 정렬

1. **1순위 (실력):** 시즌 내 평균 그로스 스코어 낮은 순
2. **2순위 (성실성):** 평균 그로스 동점 시 출전 횟수 많은 순 (백카운트)

### 4.4. AI 스코어 추출 표준 프롬프트

관리자가 스코어카드 이미지를 AI에 입력할 때 사용하는 표준 프롬프트입니다.  
관리자 화면 3단계의 **"📋 표준 프롬프트 복사"** 버튼으로 클립보드에 복사됩니다.

```
첨부하는 골프 스코어카드 이미지를 보고, 선수 4명의 18홀 오버파 타수를 추출해서 JSON 형식으로 만들어줘.

[규칙]
1. 전반 9홀과 후반 9홀의 타수를 합쳐서 선수당 정확히 18개의 숫자로 배열(Array)을 만들어.
2. 타수 표기법:
   - 파(Par)는 0, 버디는 -1(이글 -2), 오버파는 1·2·3 등 숫자만.
3. key 이름(name, scores)을 임의로 바꾸지 마.
4. 오직 JSON 코드만 출력해.

[출력 형식]
[
  {"name": "홍길동", "scores": [0,1,-1,1,2,0,1,0,1,0,0,1,1,2,0,1,0,0]},
  ...
]
```

## 5. 관리자 화면 워크플로우

### 5.1. 메인 탭 구조 (4개)

| 탭 | 주요 기능 |
|----|-----------|
| ⚙️ 대회관리 | 대회 생성·편집, 조편성, 코스매핑, 스코어입력, 신페정산, 대회종료 |
| 👤 회원관리 | 신규 회원 등록, 정보 수정, 활동상태 전환(활동중↔중지) |
| ⛳ 코스관리 | 신규 코스 등록, 사용여부 전환(Y↔N) |
| 🏆 시즌관리 | 시즌 생성, 종료 처리 |

### 5.2. 대회관리 단계 탭 (4단계)

```
1단계: 세팅&수정  →  2단계: 현장 매핑  →  3단계: 스코어 입력  →  4단계: 대회종료
```

#### 1단계 — 세팅&수정

- 대회명, 날짜, 구장, 참가 인원 설정
- 활동중 회원 명단에서 조 편성 (4명 × N조)
- 팀 배정 (백팀/청팀) 토글
- `[명단 확정 및 DB 적재]` → `tb_Record` 선수 행 사전 삽입 + `tb_Match_Episodes` 생성

#### 2단계 — 현장 매핑

- 조별 전반/후반 코스 배정 (사용여부=Y 코스만 표시)
- `[코스 매핑 저장]` → `tb_Record.전반코스` / `후반코스` 업데이트

#### 3단계 — 스코어 입력

1. **📋 표준 프롬프트 복사** → 별도 AI창에서 이미지 → JSON 변환
2. JSON 붙여넣기 → **🔍 검증 표 만들기** → 미리보기 확인
3. **💾 임시저장** → `tb_Record.H1~H18` 업데이트 (정산 컬럼 미변경)
4. **🎲 제외홀 추첨 & 신페리오 정산** → 제외홀 추첨 + NP 계산 실행

#### 4단계 — 대회종료

- 대회 정보 / 코스 / 제외홀 / 완료구분 요약 표시
- **🚩 대회 최종 마감** → `tb_Match_Episodes.완료구분 = '완료'` 저장
- 마감 완료 시 세션 드롭다운에서 제외

### 5.3. 세션 이어하기 엔진

- 화면 최상단 `[작업할 대회 선택]` 드롭다운 상시 노출
- `완료구분 != '완료'` 인 진행중 대회만 목록에 표시
- 중간에 브라우저가 종료되어도 드롭다운에서 선택 시 조편성 데이터 복원 후 작업 재개 가능

### 5.4. 회원 등록 자동채번

1. 성별 선택 (남성 / 여성)
2. 회원번호 자동 표시 (읽기전용)
