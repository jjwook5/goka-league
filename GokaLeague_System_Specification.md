# ⛳ 고카리그 (GOKA LEAGUE) 시스템 개발 명세서 Ver 1.2

본 문서는 **고카리그(GOKA LEAGUE) Ver 1.2** 의 아키텍처, 데이터베이스 설계, 비즈니스 로직 및 UI/UX 구조를 기록한 공식 명세서입니다.  
향후 기능 고도화 및 유지보수 시 본 문서를 기준으로 처리합니다.

## 1. 시스템 개요 (System Overview)

| 항목 | 내용 |
|------|------|
| **시스템명** | 고카리그 (GOKA LEAGUE) Ver 1.2 |
| **대상** | 반기별 승강제 골프 동호회 운영 관리자 및 회원 |
| **배포 환경** | GitHub + Vercel (정적 호스팅, 자동 배포) |
| **데이터베이스** | Supabase (PostgreSQL) |
| **백엔드 미들웨어** | **없음** — 프론트엔드가 Supabase REST API 직접 호출 |
| **회원 접속 URL** | `golfriend.vercel.app` |
| **관리자 접속 URL** | `golfriend.vercel.app/admin/` |

### 1.1. 주요 핵심 기능

1. **시즌제 승강제 연동** — 상/하반기 단위 독립 시즌 운영 및 1부/2부/3부 리그 관리
2. **AI 스코어 추출 연동** — 스코어카드 이미지를 AI가 분석하여 JSON 변환 → 붙여넣기 기반 입력
3. **신페리오 정산 엔진** — 무작위 제외홀 추첨 + 소수 둘째자리 정밀 핸디캡 연산 (스마트스코어 기준)
4. **운세 분석 (Luck Factor)** — 추첨 운에 의한 변수를 실력 점수와 비교 계량화
5. **관리자 통합 관리** — 회원/코스/시즌/대회 CRUD를 관리자 화면에서 직접 처리
6. **모바일 퍼스트 PWA** — 홈 화면 설치 시 네이티브 앱처럼 동작 (주소창 없음)

### 2.4. Supabase REST API 호출 패턴

```javascript
// 공통 헤더 설정
const SB_H  = { 'apikey': KEY, 'Authorization': 'Bearer ' + KEY };
const SB_HW = { ...SB_H, 'Content-Type': 'application/json', 'Prefer': 'return=minimal' };

// 조회 (GET)
fetch(`${SUPABASE_URL}/rest/v1/{테이블}?select=*{필터}`, { headers: SB_H })

// 삽입 (POST)
fetch(`${SUPABASE_URL}/rest/v1/{테이블}`, { method:'POST', headers: SB_HW, body: JSON.stringify(rows) })

// 수정 (PATCH)
fetch(`${SUPABASE_URL}/rest/v1/{테이블}?{필터}`, { method:'PATCH', headers: SB_HW, body: JSON.stringify(fields) })

// 삭제 (DELETE)
fetch(`${SUPABASE_URL}/rest/v1/{테이블}?{필터}`, { method:'DELETE', headers: SB_H })

// 한국어 필터값은 반드시 encodeURIComponent() 적용
// 예: `전화번호=eq.${encodeURIComponent('010-1000-0001')}`
```

## 3. 데이터베이스 설계 (Supabase PostgreSQL)

### 3.1. `tb_member` — 회원 마스터

| 컬럼명 | 타입 | 설명 |
|--------|------|------|
| `순번` | int | 자동 증가 |
| `전화번호` | text | **PK** — 실제로는 회원번호로 사용 |
| `성명` | text | 실명 |
| `닉네임` | text | 선택 입력 |
| `소속리그` | text | 1부 / 2부 / 3부 |
| `등급` | text | 일반 / 스텝 |
| `활동상태` | text | **활동중** / 중지 (소프트 삭제) |

**회원번호 채번 규칙** (필드명은 `전화번호` 유지)

| 성별 | 형식 | 예시 |
|------|------|------|
| 남성 | `010-1000-XXXX` | `010-1000-0001` |
| 여성 | `010-2000-XXXX` | `010-2000-0002` |

- 뒷 4자리(XXXX)는 **남녀 공통 최댓값 + 1** (zero-padding 4자리)
- 관리자 화면에서 성별 선택 시 자동 채번 표시 (수동 입력 불가)

### 3.2. `tb_Course` — 코스 마스터

| 컬럼명 | 타입 | 설명 |
|--------|------|------|
| `구장명` | text | **PK(복합)** |
| `코스명` | text | **PK(복합)** |
| `P1`~`P9` | int | 홀별 기준타수 (3 / 4 / 5) |
| `사용여부` | text | **Y** / N (소프트 삭제) |

- `사용여부 = 'Y'` 인 코스만 대회 매핑 화면 및 신규 대회 생성 시 표시
- 코스 삭제 시 물리 삭제 없이 `사용여부 = 'N'`으로 처리하여 과거 기록 무결성 유지

### 3.3. `tb_Match_Episodes` — 대회 에피소드

| 컬럼명 | 타입 | 설명 |
|--------|------|------|
| `대회번호` | bigint | `YYYYMMDD` 형식 날짜 채번 |
| `대회명` | text | 대회 이름 |
| `날짜` | date | 라운딩 날짜 |
| `구장명` | text | 사용 골프장 |
| `코스` | text | 9홀 코스명 (전반/후반 각각 1행) |
| `제외홀` | text | 신페리오 추첨 결과 (쉼표 구분, 예: `2,5,7`) |
| `완료구분` | text | **진행중** / 마감 / 완료 (3단계) |

- 하나의 대회에 코스 수만큼 행이 생성 (일반적으로 구장 내 코스 수 = 2~3행)

| 단계 | 관리자 세션 드롭다운 | 사용자 대회 콤보박스 | 신페리오/번외/청백전 |
|------|------|------|------|
| **진행중** | 표시됨 | **미표시** | - |
| **마감** | 표시됨 `[마감]` | 표시됨 | 🔒 잠금 |
| **완료** | **미표시** | 표시됨 | 전체 공개 |

## 4. 핵심 비즈니스 로직

### 4.1. 신페리오 제외홀 추첨

- **실행 시점:** 관리자 화면 → 대회관리 → 3단계 스코어 입력 페이지의 **🎲 제외홀 추첨 & 신페리오 정산** 버튼
- **전제 조건:** 해당 대회의 모든 선수 `합계 ≥ 1` 입력 완료 시에만 정산 실행 (미입력 선수 있으면 안내 후 중단)
- **알고리즘:** `tb_Course` 마스터에서 코스별 P3/P4/P5 홀 번호 분류 후, 파별 각 1개 무작위 추출 (9홀당 3개 = 18홀 총 6개 제외홀)
- **저장:** `tb_Match_Episodes.제외홀`에 쉼표 구분 문자열로 저장
- **재추첨 가능:** 대회 완료 전까지 버튼을 여러 번 눌러 재추첨 가능
- **🎲 버튼 표시:** 스코어 임시저장 후에도 항상 표시 (별도 영역으로 분리)

### 4.2. 신페리오 핸디캡 (NP Engine)

모든 계산 로직은 **브라우저 측 JavaScript**에서 실행됩니다 (서버 미들웨어 없음).  
**스마트스코어 앱 기준** — 버디 이하 실제 음수값 그대로 사용 (언더파 캡처리 없음).

```
① 제외홀 적용
   선택된 12개 홀의 오버파 실제값 합산 (버디=-1, 파=0, 보기=1 등 그대로)
   overParSelected = 추첨된 12개 홀의 실제 오버파 합산

② NP_Handicap 공식
   NP_Handicap = Math.round(overParSelected × 1.5 × 0.8 × 100) / 100
                 → 소수 둘째자리

③ 그로스 스코어
   합계 = 코스 기준타수 합산 + 18홀 오버파 실제값 합산
   ※ 버디(-1), 이글(-2) 등 언더파는 보정 없이 실제 음수 그대로 합산
   ※ 임시저장 시 합계 자동 계산하여 저장

④ Net_Score
   Net_Score = Math.round((합계 - NP_Handicap) × 100) / 100
               → 소수 둘째자리

⑤ Luck_Factor
   Luck_Factor = Math.round((NP_Handicap - (합계 - 기준타) × 0.8) × 100) / 100
                 → 소수 둘째자리
                 양수: 행운아 (핸디캡을 많이 받음)
                 음수: 불운왕 (핸디캡을 적게 받음)
```

> **Ver 1.1 대비 수정사항:**  
> - 버디 캡처리(`Math.max(0, score)`) 제거 — 실제 음수값 그대로 사용  
> - Luck_Factor 공식: `overParTotal(capped) × 0.8` → `(합계 - 기준타) × 0.8`

### 4.3. 동점자 처리 규정

순위는 저장된 필드 없이 **렌더링 시점에 JavaScript로 실시간 계산**됩니다.  
모든 화면(관리자·사용자)에 동일한 기준이 적용됩니다.

#### 신페리오 순위

| 순서 | 기준 | 방향 |
|------|------|------|
| 1차 | Net_Score | 낮은 순 |
| 2차 | 합산(Gross) | **높은 순** ← 재미 요소 (같은 Net에서 더 많이 친 사람 우선) |
| 3차 | 백카운트 | 후반9홀 → 후반8홀 → ... → H18 단독 (낮은 순) |

#### 스트로크 순위

| 순서 | 기준 | 방향 |
|------|------|------|
| 1차 | 합산(Gross) | 낮은 순 |
| 2차 | 백카운트 | 후반9홀 → 후반8홀 → ... → H18 단독 (낮은 순) |

#### 백카운트 알고리즘

```
후반9홀 합계(H10~H18) 비교
  └ 같으면 → 후반8홀(H11~H18) 비교
       └ 같으면 → 후반7홀(H12~H18) 비교
            └ ...
                 └ 같으면 → H18 단독 비교
                      └ 완전 동일 → 공동순위 부여
```

### 4.4. 리그순위 정렬 기준

| 순서 | 기준 | 방향 |
|------|------|------|
| 1차 | 평균 그로스 타수 | 낮은 순 |
| 2차 | 시즌 참가횟수 | 많은 순 |
| 3차 | 최근 대회 성적 | 낮은 순 (가장 최근 대회의 합계) |

> 시즌 내 대회를 날짜(대회번호) 오름차순 정렬 후 집계하여 "최근 대회"를 정확히 판별.

### 4.5. AI 스코어 추출 표준 프롬프트

관리자가 스코어카드 이미지를 AI에 입력할 때 사용하는 표준 프롬프트입니다.  
관리자 화면 3단계의 **"📋 표준 프롬프트 복사"** 버튼으로 클립보드에 복사됩니다.

```
첨부하는 골프 스코어카드 이미지를 보고, 선수 4명의 18홀 오버파 타수를 추출해서 JSON 형식으로 만들어줘.

[규칙]
1. 전반 9홀과 후반 9홀의 타수를 합쳐서 선수당 정확히 18개의 숫자로 배열(Array)을 만들어.
2. 타수 표기법:
   - 파(Par)는 0, 버디는 -1(이글 -2), 오버파는 1·2·3 등 숫자만.
3. key 이름(name, scores)을 임의로 바꾸지 마.
4. 오직 JSON 코드만 출력해.

[출력 형식]
```

## 5. 관리자 화면 워크플로우

### 5.1. 메인 탭 구조 (4개)

| 탭 | 주요 기능 |
|----|-----------|
| ⚙️ 대회관리 | 대회 생성·편집, 조편성, 코스매핑, 스코어입력, 신페정산, 결과조회 |
| 👤 회원관리 | 신규 회원 등록, 정보 수정, 활동상태 전환(활동중↔중지) |
| ⛳ 코스관리 | 신규 코스 등록, 사용여부 전환(Y↔N) |
| 🏆 시즌관리 | 시즌 생성, 종료 처리 |

### 5.2. 대회관리 단계 탭 (4단계)

```
1단계: 세팅&수정  →  2단계: 현장 매핑  →  3단계: 스코어 입력  →  4단계: 결과조회
```

#### 1단계 — 세팅&수정

- 대회명, 날짜, 구장, 참가 인원 설정
- **대기명단:** 활동중 회원을 **닉네임 오름차순**으로 정렬 표시 (닉네임 없으면 성명 기준)
  - 칩 표시 형식: `닉네임 (성명) 소속리그`
- 대기명단에서 선택 → 조 편성 (4명 × N조), 조 표시는 성명 사용
- 팀 배정 (백팀/청팀) 토글
- `[명단 확정 및 DB 적재]` → `tb_Record` 선수 행 사전 삽입 + `tb_Match_Episodes` 생성

#### 2단계 — 현장 매핑

- 조별 전반/후반 코스 배정 (사용여부=Y 코스만 표시)
- **대회 선택 시 기저장된 전반/후반 코스 자동 복원** (드롭다운 selected 적용)
- `[코스 매핑 저장]` → `tb_Record.전반코스` / `후반코스` 업데이트

#### 3단계 — 스코어 입력

1. **📋 표준 프롬프트 복사** → 별도 AI창에서 이미지 → JSON 변환
2. JSON 붙여넣기 → **🔍 검증 표 만들기** → 미리보기 확인
3. **💾 임시저장** → `tb_Record.H1~H18` + **합계(그로스) 자동 계산** 업데이트
4. **🎲 제외홀 추첨 & 신페리오 정산** — 항상 표시 (임시저장 후에도 유지)
   - 모든 선수 `합계 ≥ 1` 미충족 시: 안내 메시지 표시 후 중단

#### 4단계 — 결과조회

**상단 — 마감 버튼**
- **🚩 마감** → `tb_Match_Episodes.완료구분 = '마감'` 저장
  - 사용자 화면 대회 콤보박스에 노출 시작
  - 신페리오/번외타이틀/청백전은 잠금(🔒) 상태 (스트로크만 공개)

**중단 — 결과 서브탭 (3개)**

| 서브탭 | 내용 |
|--------|------|
| 신페리오 | Net_Score 기준 순위 (동점자 처리 규정 적용), 금/은/동 색상 강조 |
| 번외타이틀 | 버디/파/보기/더블 집계, **헤더 클릭 시 해당 기준 정렬** + 강조 표시 |
| 청백전 | 청팀/백팀 대결 구도, 팀 합계, 승리팀 👑 표시 |

**하단 — 결과공개 버튼**
- **🏆 결과공개** → `tb_Match_Episodes.완료구분 = '완료'` 저장
  - 모든 탭(신페리오/번외타이틀/청백전) 사용자 공개
  - 관리자 세션 드롭다운에서 제외

### 5.3. 세션 이어하기 엔진

- 화면 최상단 `[작업할 대회 선택]` 드롭다운 상시 노출
- `완료구분 != '완료'`인 진행중 대회만 목록에 표시 (`[마감]` 레이블 병기)
- 중간에 브라우저가 종료되어도 드롭다운에서 선택 시 조편성 데이터 복원 후 작업 재개 가능

### 5.4. 회원 등록 자동채번

1. 성별 선택 (남성 / 여성)
2. 회원번호 자동 표시 (읽기전용)
   - 기존 전체 회원의 뒷 4자리 최댓값 + 1로 채번
3. 성명, 닉네임, 소속리그, 등급 입력 후 등록

- `tb_Course`: PK = `(구장명, 코스명)` 복합키, `사용여부` 컬럼 추가
- `tb_Record`: PK = `(대회번호, 전화번호)` 복합키
- `tb_Season`: `종료_대회번호` bigint 타입으로 정규화 (NULL = 진행중)

**관리자 화면 전면 개편**
- 4개 메인 탭: 대회관리 / 회원관리 / 코스관리 / 시즌관리
- 대회관리 4단계 구조 (세팅 → 매핑 → 스코어 → 종료)
- 회원 등록: 성별 선택 + 자동채번 (010-1000/2000-XXXX)

**신페리오 엔진 고도화**
- 계산 로직 GAS(서버)에서 브라우저 JavaScript로 이전
- NP_Handicap, Net_Score, Luck_Factor 소수 둘째자리 정밀도
